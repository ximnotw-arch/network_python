## 🧪 Что проверяют тесты

| Вид проверки | Пример |
|-------------|--------|
| **Корректность** | Результаты совпадают с ожидаемыми |
| **Конкурентность** | Код действительно работает параллельно (замер времени) |
| **Граничные случаи** | Пустой список, один элемент |
| **Потокобезопасность** | Гонок данных нет (Lock) |
| **Производительность** | Параллельная версия быстрее последовательной |
| **Таймауты** | Операция прерывается по истечении времени |
| **Отмена** | CancelledError обрабатывается корректно |
| **Групповые ошибки** | TaskGroup правильно собирает исключения |

## 📋 Описание заданий

### 01_threadpool — Многопоточная загрузка данных

**Контекст:** сервис собирает данные из нескольких внешних API. Каждый HTTP-запрос занимает ~50 мс — это I/O-bound задача, где процессор простаивает в ожидании ответа.

- **1.1** — Реализовать загрузку нескольких URL через пул потоков. Результаты должны быть в том же порядке, что и URL.
- **1.2** — Некоторые URL могут быть недоступны. Нужно обработать ошибки: если URL содержит "bad", имитировать ошибку соединения и вернуть None для такого URL. Остальные URL должны быть обработаны независимо.
- **1.3** *(повышенная сложность)* — Добавить уведомление о прогрессе. После завершения каждого URL вызывать progress_callback(completed, total). Результаты возвращать по мере завершения, а не в исходном порядке.

### 02_race_condition — Гонка данных и Lock

**Контекст:** несколько потоков одновременно изменяют общий счётчик. Без синхронизации часть изменений теряется.

- **2.1** — Реализовать функцию, которая создаёт race condition: чтение и запись счётчика не атомарны, между ними есть искусственная задержка.
- **2.2** — Исправить гонку с помощью Lock. Критическая секция должна быть минимальной (только чтение + запись, не весь цикл).
- **2.3** *(повышенная сложность)* — Реализовать потокобезопасный банковский счёт с методами deposit, withdraw, get_balance. При нехватке средств withdraw выбрасывает InsufficientFundsError.

### 03_multiprocessing — Многопроцессорные вычисления

**Контекст:** нужно вычислить сумму простых чисел для каждого элемента списка. Это CPU-bound задача — потоки не ускорят из-за GIL.

- **3.1** — Реализовать последовательную и параллельную (через multiprocessing.Pool) версии вычислений.
- **3.2** — Реализовать ту же задачу через ThreadPoolExecutor — должно быть медленнее, чем Pool. Это демонстрация влияния GIL.

### 04_asyncio — Асинхронное программирование (7 заданий)

**Контекст:** нужно обработать тысячи запросов в одном потоке. Asyncio позволяет держать тысячи соединений без создания потоков.

- **4.1** — Написать первую корутину: async def функция с await asyncio.sleep().
- **4.2** — Запустить несколько корутин конкурентно через asyncio.gather().
- **4.3** — Использовать asyncio.TaskGroup для запуска группы задач и обработки исключений.
- **4.4** — Добавить таймаут для долгой операции через asyncio.wait_for().
- **4.5** — Реализовать корректную отмену задачи: отловить CancelledError и подчистить ресурсы.
- **4.6** — Использовать asyncio.as_completed() для получения результатов по мере готовности.
- **4.7** *(повышенная сложность)* — Выгрузить CPU-bound код в пул потоков через asyncio.to_thread(), чтобы не блокировать event loop.
